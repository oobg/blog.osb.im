# Node.js 의 소개와 특징

## Node.js 란?
Node.js 는 아래와 같이 3가지로 요약할 수 있다.

- Chrome V8 JavaScript 엔진으로 빌드된 JavaScript 런타임이다.
- 이벤트 기반, 논 블로킹 I/O 모델을 사용해 가볍고 효율적이다.
- 패키지 생태계인 `npm(Node Package Manager) 을 가지고 있다. 이는 세계에서 가장 큰 오픈 소스 라이브러리 생태계이다.

이제 위 내용에 대해 자세히 알아보자.

## Chrome V8 엔진

Node.js 를 설명하기에 앞서 Chrome V8 엔진에 대해 알아야 한다.<br />
Node.js 는 Chrome V8 JavaScript 엔진으로 빌드되었기 때문이다.

Chrome V8 JavaScript 엔진은 Google 에서 개발한 오픈 소스 고성능 JavaScript 및 WebAssembly 엔진이다.<br />
이 엔진은 빠른 실행 속도를 자랑하며, Node.js 뿐만 아니라 브라우저에서도 사용된다.

### 특징

다음은 V8 엔진에 대한 주요 특징이다.

- **빠른 실행 속도**: V8 엔진은 Just-In-Time (JIT) 컴파일을 사용하여 JavaScript 코드를 기계어로 변환하여 빠르게 실행한다.
- **메모리 관리**: V8은 효율적인 가비지 컬렉션을 통해 메모리 관리를 최적화한다.
- **최신 JavaScript 기능 지원**: ECMAScript 표준의 최신 기능을 빠르게 지원한다.
- **크로스 플랫폼**: V8은 다양한 운영 체제에서 동작하며, Node.js와 같은 서버 측 환경에서도 사용된다.

### 예제

다음은 V8 엔진을 사용하는 간단한 예제이다.

```javascript
// V8 엔진을 사용하는 Node.js 예제
const { exec } = require("child_process");

exec("node -v", (error, stdout, stderr) => {
  if (error) {
    console.error(`Error: ${error.message}`);
    return;
  }
  if (stderr) {
    console.error(`Stderr: ${stderr}`);
    return;
  }
  console.log(`Node.js version: ${stdout}`);
});
```

이 예제는 Node.js의 버전을 출력하는 간단한 스크립트다.<br />
V8 엔진 덕분에 빠르고 효율적으로 실행된다.


## Javascript 런타임

Javascript 런타임이란 무엇을 의미할까?

JavaScript 런타임은 JavaScript 코드를 실행할 수 있는 환경을 의미합니다. 일반적으로 브라우저와 Node.js가 대표적인 JavaScript 런타임 환경입니다.

### 주요 특징

- **이벤트 루프**: JavaScript 런타임은 이벤트 루프를 사용하여 비동기 작업을 처리합니다. 이는 논 블로킹 I/O를 가능하게 하여 높은 성능을 제공합니다.
- **콜 스택**: JavaScript 런타임은 콜 스택을 사용하여 함수 호출을 관리합니다. 콜 스택은 함수가 호출될 때마다 쌓이고, 함수가 반환될 때마다 제거됩니다.
- **힙**: 메모리 할당을 위한 구조로, 객체와 변수를 저장합니다.

### 브라우저 런타임

브라우저 런타임은 웹 페이지에서 JavaScript 코드를 실행하는 환경입니다. 주요 구성 요소는 다음과 같습니다:

- **DOM (Document Object Model)**: HTML 문서를 객체로 표현하여 JavaScript가 이를 조작할 수 있게 합니다.
- **BOM (Browser Object Model)**: 브라우저 창과 관련된 객체를 제공합니다.
- **Web APIs**: `fetch`, `localStorage`, `setTimeout` 등 다양한 API를 제공합니다.

### Node.js 런타임

Node.js 런타임은 서버 측에서 JavaScript 코드를 실행하는 환경입니다. 주요 구성 요소는 다음과 같습니다:

- **V8 엔진**: Google의 V8 엔진을 사용하여 JavaScript 코드를 빠르게 실행합니다.
- **내장 모듈**: `fs`, `http`, `path` 등 다양한 내장 모듈을 제공합니다.
- **npm**: 패키지 매니저로, 다양한 오픈 소스 라이브러리를 쉽게 설치하고 관리할 수 있습니다.

### 예제

다음은 Node.js 런타임에서 간단한 HTTP 서버를 만드는 예제입니다:

```javascript
const http = require("http");

const hostname = "127.0.0.1";
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader("Content-Type", "text/plain");
  res.end("Hello, World!\n");
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

이 코드를 실행하면 http://127.0.0.1:3000/ 에서 "Hello, World!" 메시지를 확인할 수 있습니다.


## Node.js 의 장점

1. **빠른 성능**: Node.js 는 V8 엔진을 사용하여 빠른 실행 속도를 자랑한다.
2. **확장성**: 이벤트 기반 아키텍처 덕분에 확장성이 뛰어납니다.
3. **커뮤니티 지원**: 활발한 커뮤니티와 풍부한 모듈이 존재합니다.

## Node.js 의 단점

1. **싱글 스레드**: CPU 집약적인 작업에 적합하지 않습니다.
2. **콜백 지옥**: 비동기 프로그래밍으로 인해 콜백 지옥에 빠질 수 있습니다.

## 주요 사용 사례

- **웹 서버**: Express.js 와 같은 프레임워크를 사용하여 빠르고 효율적인 웹 서버를 구축할 수 있습니다.
- **API 서버**: RESTful API 서버를 쉽게 구축할 수 있습니다.
- **실시간 애플리케이션**: WebSocket 을 사용한 실시간 채팅 애플리케이션 등에 적합합니다.
